# Site Structure And Sitemap

Статус: `draft-for-site-v1`

## 1. Рекомендация по сайту v1

Для первого публичного релиза не стоит строить большой многостраничный
маркетинговый сайт.

Рекомендация:

```text
1 strong homepage
+ 3 lightweight supporting pages
```

Это даст хороший баланс между серьезным образом продукта и скоростью запуска.

## 2. Состав сайта v1

Рекомендуемый набор страниц:

1. `Home`
2. `Platform`
3. `Agents`
4. `Pilot / Contact`

Если нужно стартовать еще компактнее, можно выпустить `Home` как основную
страницу и сделать остальные разделы как anchor-блоки на одной странице.

## 3. Роль каждой страницы

### 3.1 Home

Главная narrative-страница сайта.

Задача:

- объяснить, что это за продукт;
- показать, чем он отличается от chatbot-only решений;
- продемонстрировать широту платформы;
- дать первый proof point;
- привести пользователя к пилоту или демо.

### 3.2 Platform

Страница про платформу как продуктовый foundation layer.

Задача:

- раскрыть основные платформенные возможности на языке бизнеса;
- объяснить, почему единая платформа лучше набора разрозненных AI-решений;
- показать serious / controlled / scalable image.

### 3.3 Agents

Страница про типы AI-агентов и сценарии использования.

Задача:

- показать, что платформа поддерживает не один use case;
- раскрыть основные archetype-направления;
- связать use cases с реальными бизнес-задачами.

### 3.4 Pilot / Contact

Страница про запуск первого сценария.

Задача:

- объяснить, с чего начинается внедрение;
- показать operator-led подход и controlled rollout;
- собрать входящие запросы.

## 4. Навигация сайта

## 5. Структура homepage

Ниже рекомендуемая последовательность секций на главной странице.

## 5.1 Hero

Цель:

- сразу назвать категорию продукта;
- дать сильное платформенное позиционирование;
- задать serious B2B tone.

Смысл:

`AI Agents Platform for Business Operations`

## 5.2 Why Now / Market Shift

Цель:

- показать переход от ручных операций и разовых AI-интеграций к
  AI-first operations;
- создать ощущение своевременности.

## 5.3 What The Platform Does

Цель:

- объяснить продукт как единую управляемую платформу;
- не уходить в низкоуровневую архитектуру;
- показать, что здесь уже есть agent management, knowledge, channels, control
  and scalability foundation.

## 5.4 Agent Categories

Цель:

- показать широту платформы через 5-6 agent categories;
- уйти от восприятия "это один чат-бот".

Рекомендуемые карточки:

- Sales Agent
- Support Agent
- Request Processing Agent
- Operations Agent
- Back-office Assistant
- Knowledge / Compliance Agent

## 5.5 First Service Proof

Цель:

- дать конкретный и понятный сценарий первого запуска;
- показать, что платформа не абстрактна;
- связать marketing message с реальным productization flow.

Рекомендуемый фокус:

`Website Sales & Support Knowledge Agent`

## 5.6 Why It Is Not Just A Bot

Цель:

- объяснить отличия от chatbot builder и prompt-wrapper решений;
- усилить платформенный образ.

## 5.7 Trust, Control And Readiness

Цель:

- показать зрелость;
- дать основания доверять платформе как базе для бизнеса.

Основные темы:

- tenant isolation;
- controlled knowledge;
- safe public channel;
- observability;
- traceability;
- release discipline.

## 5.8 How Pilot Launch Works

Цель:

- снять тревогу про "как это внедрять";
- показать controlled first step;
- приблизить конверсию.

Рекомендуемый flow:

1. Определяем сценарий.
2. Настраиваем агента и знания.
3. Подключаем канал.
4. Запускаем пилот.
5. Анализируем качество и расширяем use cases.

## 5.9 Final CTA

Цель:

- завершить narrative понятным действием;
- конвертировать интерес в контакт.

4. What the platform team sets up
5. What success looks like
6. Contact / request form

## 9. Что можно отложить на later phase

На первый релиз можно не выносить в отдельные страницы:

- pricing;
- deep technical architecture;
- portal UI walkthrough;
- detailed API docs;
- marketplace vision как самостоятельный раздел.

Это можно добавить позже после первых пилотов и product proof.

## 10. Итог

Для v1 сайт должен быть компактным, но не маленьким по восприятию.

Правильная модель:

- одна сильная homepage;
- несколько supporting pages;
- четкий B2B narrative;
- platform-first messaging;
- concrete first-service proof.
