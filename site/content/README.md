# Site Content Pack

Этот раздел хранит контентную основу для первого публичного сайта платформы.

Цель пакета:

- зафиксировать фокус и позиционирование сайта;
- определить рекомендуемую структуру страниц и секций;
- подготовить готовые тексты для первого публичного релиза;
- дать reusable-копирайтинг для CTA, навигации и коротких блоков.

Во всех документах ниже используется рабочее имя `Core AI Platform`.
Если для публичного запуска утверждено отдельное бренд-имя, его можно заменить
в текстах без изменения общей структуры и логики позиционирования.

## Состав пакета

- [01_SITE_FOCUS_AND_POSITIONING_RU.md](01_SITE_FOCUS_AND_POSITIONING_RU.md):
  фокус сайта, целевая аудитория, рекомендованная модель позиционирования и
  content principles.
- [02_SITE_STRUCTURE_AND_SITEMAP_RU.md](02_SITE_STRUCTURE_AND_SITEMAP_RU.md):
  структура сайта, состав v1-страниц и логика homepage по секциям.
- [03_HOMEPAGE_COPY_RU.md](03_HOMEPAGE_COPY_RU.md):
  готовые тексты для главной страницы: hero, блоки, use cases, proof,
  trust-sections и финальные CTA.
- [04_CTA_AND_SHORT_COPY_RU.md](04_CTA_AND_SHORT_COPY_RU.md):
  короткие формулировки для кнопок, форм, навигации, meta и повторяемых
  интерфейсных фрагментов.

## Как использовать

Рекомендуемая последовательность:

1. Сначала согласовать [01_SITE_FOCUS_AND_POSITIONING_RU.md](01_SITE_FOCUS_AND_POSITIONING_RU.md).
2. Затем утвердить [02_SITE_STRUCTURE_AND_SITEMAP_RU.md](02_SITE_STRUCTURE_AND_SITEMAP_RU.md).
3. После этого использовать [03_HOMEPAGE_COPY_RU.md](03_HOMEPAGE_COPY_RU.md)
   как source of truth для дизайна и верстки homepage.
4. [04_CTA_AND_SHORT_COPY_RU.md](04_CTA_AND_SHORT_COPY_RU.md) использовать как
   библиотеку коротких текстов при сборке интерфейса.

## Роль сайта v1

Первый сайт должен:

- представлять платформу как серьезный B2B-продукт;
- показывать широту платформы, а не одного бота;
- иметь понятный первый proof point через Website Sales & Support Knowledge
  Agent;
- собирать интерес к пилоту, демо и первым внедрениям.

Первый сайт не должен:

- выглядеть как лендинг одного FAQ-бота;
- обещать неподтвержденные результаты как уже гарантированные;
- уходить в чрезмерную архитектурную или DevOps-детализацию;
- делать вид, что self-service SaaS уже полностью готов, если текущая модель
  запуска пока operator-led.
