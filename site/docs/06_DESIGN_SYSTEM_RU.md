# Дизайн-система «Контраст-дуэт» (переносимая)

Статус: `design-system`

Это описание визуального языка, в котором сделан сайт AIxDC, в **контент-независимом**
виде — чтобы по нему можно было собрать другой сайт в том же стиле. Реализация —
один рукописный `styles.css` без фреймворков и сборки (значения ниже совпадают с
`public/static/styles.css`).

## 1. Суть стиля

```text
Светлая премиум-база + полноширинные тёмные «инженерные» полосы.
Лаконично, дорого, без картинок. Серьёзный B2B/enterprise.
```

Принципы:

- **Никаких изображений.** Только типографика, тонкие линии, плашки, CSS-графика.
- **Контраст-дуэт.** Светлые секции чередуются с полноширинными тёмными графитовыми
  полосами (`band-dark`) — это главный приём ритма и «дороговизны».
- **Monospace как акцент.** Технические метки, статусы, «телеметрия», координаты —
  моноширинным; основной текст — никогда.
- **Один сдержанный акцент** (сталь-cyan). Только eyebrow, статусы, мелкие точки, тонкие
  линии. Никаких полноэкранных неоновых градиентов.
- **Hairline-эстетика.** Границы 1px, радиусы ≤ 10px, тяжёлых теней нет (одна мягкая —
  на тёмной data-панели). Много воздуха, ≤ 2 CTA в смысловом блоке.

## 2. Токены (drop-in `:root`)

```css
:root {
  /* Layout */
  --shell: 1200px;
  --gutter: clamp(20px, 5vw, 56px);
  --radius: 10px;
  --radius-sm: 7px;

  /* Светлые поверхности */
  --bg: #fbfbf9;          /* фон страницы — тёплый near-white */
  --surface: #ffffff;     /* карточки/поля */
  --surface-2: #f4f4f1;   /* альтернативная поверхность, hover */

  /* Тёмные полосы */
  --ink: #0c0f14;         /* фон band-dark */
  --ink-2: #090b10;       /* самый глубокий (финальный CTA, трейсы) */

  /* Текст */
  --text: #14181d;
  --muted: #6b7480;
  --faint: #99a0aa;

  /* Линии */
  --line: #e7e8ec;
  --line-strong: #d7dae0;

  /* Акцент — сталь-cyan, точечно */
  --accent: #2e7da6;
  --accent-strong: #25688a;
  --accent-soft: #eaf3f8;

  /* Статусы */
  --ok: #2c8a6b;
  --warn: #b7791f;

  /* Кнопки */
  --btn: #15191f;
  --btn-hover: #262b33;

  /* Контекст внутри band-dark / тёмной панели */
  --d-text: #edeff2;
  --d-muted: #969ca8;
  --d-faint: #6c7280;
  --d-line: #232831;
  --d-line-soft: #1b1f27;
  --d-surface: #12161c;
  --d-accent: #5fb7dc;   /* акцент на тёмном — светлее */
  --d-ok: #5cc9a2;       /* статус на тёмном */

  /* Шрифты */
  --sans: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
    "Segoe UI", "Noto Sans", "Noto Sans Arabic", "Noto Sans SC", Arial, sans-serif;
  --mono: ui-monospace, "SF Mono", "JetBrains Mono", Menlo, Consolas,
    "Liberation Mono", monospace;
}
```

Правило цвета: на тёмных полосах не используем светлые токены напрямую — внутри
`.band-dark` переопределяются роли (`--d-*`). Акцент на тёмном — светлее (`--d-accent`),
чем на светлом (`--accent`).

## 3. Типографика

Шрифт нейтральный sans (Inter + системные), моно — системный стек. Никаких декоративных
headline-шрифтов и condensed для текста. Должен держать кириллицу, латиницу, CJK, арабский.

| Роль | Размер | Вес | Tracking / LH | Заметки |
|---|---|---|---|---|
| H1 `.display` | `clamp(2.6rem, 5.6vw, 4.2rem)` | 600 | `-0.022em` / 1.02 | Короткое деловое позиционирование |
| H2 `.heading` | `clamp(1.7rem, 3.3vw, 2.5rem)` | 600 | `-0.018em` / 1.1 | Названия блоков |
| H3 | `1.0625rem` (17px) | 600 | `-0.005em` / 1.3 | Свойства, карточки |
| Lead | `clamp(1.06rem, 1.6vw, 1.3rem)` | 400 | / 1.5 | Цвет `--text`, до 36ch |
| Support | `1rem` | 400 | / 1.66 | `--muted`, до 56ch |
| Prose | `1rem` | 400 | / 1.7 | `--muted` |
| Eyebrow | `11px` mono | 600 | `0.16em` upper | `--accent`, с линией-tick |
| Mono-метки | `10–11px` mono | 600–700 | `0.06–0.1em` | Метки, статусы, трейсы, meta |

Eyebrow — фирменный элемент: моно, uppercase, акцентный, с короткой линией слева.

```css
.eyebrow {
  display: inline-flex; align-items: center; gap: 11px;
  font-family: var(--mono); font-size: 11px; font-weight: 600;
  letter-spacing: 0.16em; text-transform: uppercase; color: var(--accent);
}
.eyebrow::before { content: ""; width: 20px; height: 1px; background: currentColor; opacity: .55; }
.band-dark .eyebrow { color: var(--d-accent); }
```

## 4. Сетка и ритм

- **Контейнер** `.shell`: `width:100%; max-width: var(--shell); margin-inline:auto;
  padding-inline: var(--gutter);`. На него выровнены шапка, футер и контент.
- **`<main>` на всю ширину** — это обязательно, иначе тёмные полосы не будут
  full-bleed. Ширину держит только `.shell` внутри секций.
- **Секция** `.section`: вертикальный ритм `padding-block: clamp(60px, 8vw, 116px)`.
- **Чередование полос:** между двумя светлыми секциями — hairline `border-top`; тёмная
  секция (`.band-dark`) сама задаёт ритм сменой фона на `--ink`, без бордера.
- **Двухколоночные блоки** `.cols`: `minmax(0,.82fr) minmax(0,1.18fr)`, gap
  `clamp(28px,5vw,76px)` — слева заголовок, справа контент (списки/spec).
- Максимальная ширина контента ~1200px, строки текста ограничены `ch` (36/56/64).

Рекомендуемый ритм страницы (светлый/тёмный): hero(светлый + тёмная панель) → светлая →
светлая → **тёмная** → светлая → **тёмная** → светлая → **тёмная CTA**. Две тёмные полосы
подряд не ставим.

## 5. Сигнатурные компоненты

**Кнопки.** Primary — заливка `--btn` (near-black), белый текст; secondary —
прозрачная с `--line-strong`. На тёмном инвертируются (primary становится светлой плашкой
с тёмным текстом). Радиус `--radius-sm`, высота 46px, ≤ 2 в блоке.

**Тёмная data-панель `.console`** — ключевой образ «операционной системы»: тёмная карточка
(градиент `#13171e → --ink`, бордер `--d-line`, одна мягкая тень). Внутри:
- `.console-bar` — моно-id (`Brand.system`) + заголовок + `.status-pill`;
- `.signal-grid` — 2×2 definition list (dt — моно uppercase `--d-faint`, dd — `--d-text`),
  hairline-разделители; для чистоты показываем ровно 4 факта;
- `.console-trace` — моно-строки на `--ink-2`, каждая с префиксом `>` цвета `--d-accent`.

**`.status-pill`** — моно-капсула статуса с точкой и мягким свечением (`box-shadow`),
цвет `--d-ok`.

**`.spec`** — список «спецификации» (строки с hairline-разделителями и точкой `--d-accent`
слева); живёт в тёмной полосе (инфраструктура/параметры).

**`.matrix`** — карточки услуг/аудиторий в **редакционном** стиле: 3 колонки, у каждой
карточки только тонкая верхняя линия (`border-top: var(--line-strong)`), моно-индекс
(`01`, `02`…), H3, muted-текст. Без рамок-ящиков и без теней — нечётное число элементов
выглядит естественно.

**`.check-list`** — пункты с квадратной точкой-бордером `--accent` (на тёмном `--d-accent`).

**Шапка** — sticky, полупрозрачный фон + `backdrop-filter: blur`, hairline снизу; внутри
`.shell`: бренд, навигация (text-links, `--muted`→`--text` на hover), справа основная
кнопка + переключатель языков.

**Футер** — светлый, hairline сверху: бренд, ряд text-ссылок, моно-`© год Brand`.

**Финальный CTA** — самая глубокая полоса `--ink-2`, по центру: eyebrow, H2, prose,
кнопки, и моно-строка `cta-meta` (например `Brand · подзаголовок · код`).

## 6. Адаптив

- `≤ 1000px`: `.hero-grid` и `.cols` → 1 колонка; навигация скрывается (бренд + основная
  кнопка + язык остаются).
- `721–1000px`: `.matrix` → 2 колонки.
- `≤ 720px`: `.matrix` → 1 колонка; тэглайн бренда скрыт.
- `≤ 440px`: уменьшаем паддинги панели/секций.
- RTL (`dir="rtl"`): моно-фрагменты (id, trace, meta, индексы) изолируем `direction:ltr;
  unicode-bidi:isolate`. Логические свойства (`padding-inline`, `inset-inline-start`).
- `prefers-reduced-motion`: отключаем transition.

## 7. Запреты (чтобы не «расплылось»)

- Никаких фото, иконок-заглушек, 3D, частиц, неоновых градиентов на весь экран.
- Акцентный цвет не доминирует — только метки/статусы/линии.
- Моно — только техническое; не использовать для абзацев.
- Радиусы не больше 10px; тяжёлых теней нет; карточки не вкладываем друг в друга.
- Заголовки внутри карточек/таблиц — компактные (чем ближе к фактам, тем меньше).

## 8. Перенос на другой сайт

1. Скопировать `:root` (раздел 2), `--sans/--mono`, утилиты `.shell`, `.section`,
   `.band-dark`, `.eyebrow`, кнопки.
2. Сохранить каркас: `<main>` на всю ширину, контент в `.shell`, чередование
   светлых/тёмных полос.
3. Перенести сигнатурные компоненты по необходимости (console-панель и spec — самые
   «фирменные»; matrix и check-list — универсальные).
4. Контент другой — токены, типо-шкала, ритм и запреты те же.
