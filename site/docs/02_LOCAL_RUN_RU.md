# Локальный запуск сайта

Статус: `development-note`

## 1. Что это за проект

Сайт запускается как обычный PHP-проект без Node.js, Composer, сборки фронтенда
и базы данных.

Точка входа:

```text
index.php
```

Локальный роутер для встроенного PHP-сервера:

```text
router.php
```

## 2. Требования

Нужен установленный PHP. Рекомендуемая версия для разработки:

```text
PHP 8.3+
```

Проверить версию:

```bash
php -v
```

## 3. Быстрый запуск

Перейти в корень сайта:

```bash
cd /Volumes/Work/PC/ai_admin_panel/site
```

Запустить локальный сервер:

```bash
php -S 127.0.0.1:8101 router.php
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8101/
```

Важно запускать команду именно из корня проекта, где лежат `index.php` и
`router.php`.

## 4. Запуск в фоне через screen

Если нужно оставить сервер работать в фоне:

```bash
cd /Volumes/Work/PC/ai_admin_panel/site
screen -dmS ai_admin_panel_site php -S 127.0.0.1:8101 router.php
```

Проверить активные screen-сессии:

```bash
screen -ls
```

Остановить сервер:

```bash
screen -S ai_admin_panel_site -X quit
```

## 5. Проверка

Проверить главную страницу:

```bash
curl -I http://127.0.0.1:8101/
```

Проверить внутреннюю страницу:

```bash
curl -I http://127.0.0.1:8101/platform
```

Проверить статику:

```bash
curl -I http://127.0.0.1:8101/static/styles.css
```

В норме все три запроса должны вернуть `200 OK`.

## 6. Если порт занят

Если порт `8101` уже занят, можно использовать другой порт выше `8100`,
например `8102`:

```bash
php -S 127.0.0.1:8102 router.php
```

Тогда сайт будет доступен по адресу:

```text
http://127.0.0.1:8102/
```

## 7. Почему нужен router.php

`router.php` нужен для локальной разработки на встроенном PHP-сервере:

- существующие файлы из `static/`, `assets/`, `robots.txt`, `sitemap.xml`
  отдаются напрямую;
- остальные URL передаются в `index.php`;
- чистые маршруты вроде `/en/platform`, `/ru/agents`, `/es/pilot`
  работают так же, как на production-хостинге с rewrite-правилами.

Файл `.htaccess` используется Apache-хостингом, но встроенный PHP-сервер его не
читает.

## 8. Production-развертывание

Локальный запуск и production-деплой отличаются. Для сервера используйте
отдельный DevOps-runbook:

```text
docs/03_DEVOPS_DEPLOY_RU.md
```

В нем описаны Apache/Nginx-настройки, права на `storage/logs`, SMTP-переменные
окружения для формы заявки и checklist проверки после публикации.
